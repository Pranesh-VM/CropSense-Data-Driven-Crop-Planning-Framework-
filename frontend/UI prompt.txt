PROJECT NAME: CropSense - Agricultural Nutrient Management System

PURPOSE: Help Indian farmers manage soil nutrients through continuous crop cycles, with automatic rainfall tracking and nutrient depletion calculations.

TARGET USERS: 
- Rural farmers in India (primarily Tamil Nadu)
- Age: 25-60 years
- Tech literacy: Basic (can use WhatsApp)

USER WORKFLOW:
1. Farmer creates account
2. Enters soil nutrient data (N, P, K, pH)
3. Gets top 3 crop recommendations from AI
4. Selects a crop and starts monitoring cycle
5. System automatically tracks rainfall and updates nutrients in real-time
6. Farmer sees current nutrient status, warnings, and progress
7. After harvest, system calculates final nutrients
8. If nutrients are sufficient, farmer starts next cycle
9. If nutrients are too low, system recommends soil testing

AUTHENTICATION: JWT Bearer Token (in Authorization header)

AVAILABLE ENDPOINTS:

1. POST /api/auth/signup
   Body: { username, email, password, name, phone, location, latitude, longitude }
   Response: { success, farmer, token }

2. POST /api/auth/login
   Body: { username, password }
   Response: { success, farmer, token }

3. GET /api/auth/profile
   Headers: Authorization: Bearer <token>
   Response: { success, profile }

4. POST /api/rindm/get-recommendations
   Headers: Authorization: Bearer <token>
   Body: { N, P, K, ph, latitude, longitude }
   Response: { success, recommendation_id, recommendations: { predicted_crop, confidence, crop_1, crop_2, crop_3 }, nutrients, weather }

5. POST /api/rindm/start-cycle
   Headers: Authorization: Bearer <token>
   Body: { recommendation_id, selected_crop, N, P, K, ph, soil_type }
   Response: { success, cycle_id, crop, start_date, expected_end_date, duration_days, current_nutrients, crop_requirements }

6. GET /api/rindm/active-cycle
   Headers: Authorization: Bearer <token>
   Response: { success, has_active_cycle, cycle: { cycle_id, status, crop, progress, current_nutrients, nutrient_status, rainfall_events } }

7. GET /api/rindm/cycle-status/{cycle_id}
   Headers: Authorization: Bearer <token>
   Response: { success, cycle_id, status, crop, progress, current_nutrients, nutrient_status }

8. GET /api/rindm/history
   Headers: Authorization: Bearer <token>
   Response: { success, cycles: [...] }

9. POST /api/rindm/complete-cycle/{cycle_id}
   Headers: Authorization: Bearer <token>
   Response: { success, final_nutrients, depletion_summary, below_threshold, can_continue }
